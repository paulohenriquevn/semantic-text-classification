# Chapter 7: Conclusion

## 7.1 Summary of Findings

This thesis investigated whether a hybrid cascaded architecture combining BM25 lexical retrieval, frozen dense embeddings, and a deterministic semantic rule engine outperforms isolated paradigms in the task of intent classification over PT-BR customer service conversations. Four hypotheses were tested on a post-audit dataset of 2,122 labeled conversation windows spanning 8 intent classes, evaluated under five random seeds (13, 42, 123, 2024, 999) and Wilcoxon signed-rank significance testing. Two hypotheses were confirmed, one was inconclusive, and one was refuted.

**H1 — Hybrid Retrieval Surpasses Lexical-Only Retrieval (Confirmed, p = 0.017).** The best hybrid configuration (Hybrid-LINEAR, alpha = 0.30, weighting BM25 score at 30% and ANN score at 70%) achieved MRR = 0.853 against BM25-base at MRR = 0.835. The Wilcoxon signed-rank test returned p = 0.0165 with bootstrap 95% CI = [0.0028, 0.0334], confirming that the improvement is statistically significant and excludes zero. The effect size was r = 0.29, classified as small-to-medium under Cohen's conventions. The hybrid configuration also significantly outperformed pure ANN retrieval (MRR = 0.824, p = 0.030, 95% CI = [0.0028, 0.0552]), confirming genuine complementarity rather than redundancy: BM25 recovers exact-term matches that the encoder misses; the encoder recovers paraphrases that BM25 misses. No statistically significant differences were found across fusion weight sweeps once alpha fell below 0.65, nor between linear fusion and RRF, suggesting the system is robust to moderate hyperparameter variation once lexical weight is kept low. H1 is confirmed.

**H2 — Lexical + Embedding Features Outperform Lexical-Only Features (Confirmed, p < 10^-35).** Combining lexical TF-IDF features with 384-dimensional frozen embeddings (paraphrase-multilingual-MiniLM-L12-v2) in a LightGBM classifier yielded Macro-F1 = 0.722, compared to 0.551 for lexical-only LightGBM and 0.334 for lexical-only logistic regression. The Wilcoxon test for lexical+emb LightGBM versus lexical LightGBM returned p = 2.4 × 10^-35 with effect size r = 0.84 (large), and bootstrap 95% CI = [0.34, 0.44], unambiguously excluding zero. Gains were consistent across all 8 evaluated intent classes: per-class analysis found significant improvements in 16 of 16 tested comparisons (100%), exceeding the pre-specified criterion of 60%. The largest per-class gains were on intents that resist lexical enumeration: cancelamento (+0.66 F1 over lexical-only), suporte tecnico (+0.54), elogio (+0.47), and reclamacao (+0.38). Intents with high lexical regularity — saudacao (+0.10) — showed the smallest but still positive gains. This pattern is consistent with the theoretical expectation that dense embeddings benefit semantically diffuse classes most. The best LightGBM model trains in approximately 6 seconds on CPU, reinforcing the practical viability of the approach. H2 is confirmed.

**H3 — Rules Complement ML Classification (Inconclusive, p = 0.131).** The rules-as-features strategy (appending binary rule activation vectors to the LightGBM feature space) achieved Macro-F1 = 0.740 against 0.722 for ML-only, a difference of +0.018. The Wilcoxon test returned p = 0.131 with bootstrap 95% CI = [-0.004, 0.039], which includes zero. The observed positive direction is consistent with a genuine marginal benefit, but the sample size and the sparse rule coverage (only two lexical rules: rule_cancel and rule_complaint) are insufficient to reject the null hypothesis at alpha = 0.05. The rules-as-override strategy performed substantially worse (Macro-F1 = 0.648, p = 8.2 × 10^-5 against rules-as-features), confirming that deterministic rule hard-overrides corrupt ML predictions outside rule coverage zones and should not be used as primary decision makers. The rules-only baseline (Macro-F1 = 0.137) confirms that the 2-rule configuration is far too sparse for standalone use. H3 is inconclusive: the positive trend is encouraging, but the current rule set lacks the coverage needed to demonstrate significance.

**H4 — Cascaded Inference Reduces Cost Without Accuracy Loss (Refuted).** The H4 experiment implemented a two-stage cascade in which windows classified by a lightweight LightGBM model (lexical features only) with high confidence above a threshold are not escalated to the full pipeline (embedding + lexical features). Cascade variants at thresholds from 0.50 to 0.90 all failed to produce statistically significant improvement over the uniform baseline (all p > 0.08). More importantly, cost measurement revealed a structural failure: because the full pipeline pre-computes embeddings during feature extraction, windows that reach Stage 2 incur the full embedding cost regardless. The "cheap" Stage 1 cost per window (0.091 ms) exceeded the "full" Stage 2 cost per window (0.083 ms) in the current implementation, inverting the cost differential that is the prerequisite for cascade benefit. H4 is refuted. The cascade design as implemented does not achieve its cost-reduction objective within the constraints of a shared pre-computation architecture.

The ablation study quantifies the relative contribution of each signal family to the full pipeline's Macro-F1 = 0.740: embedding features account for +33.0 percentage points (the dominant contributor by far), lexical features for +2.9 pp, rule activation features for +1.8 pp, and structural conversation features for +1.3 pp. These figures confirm that the embedding representation is the load-bearing component of the classification pipeline.

## 7.2 Contributions

This thesis makes five distinct contributions, spanning empirical evidence, methodological decisions, and system architecture.

**C1: Empirical Evidence of Hybrid Retrieval Complementarity in the Conversational Domain.** The confirmation of H1 adds a PT-BR customer service data point to the growing literature on hybrid lexical-semantic search (Formal et al., 2021; Ma et al., 2022). The key finding is that complementarity holds even when the conversational text is short, noisy, and style-shifted relative to the encoder's training distribution. The effect size is small (r = 0.29), and practitioners should not expect dramatic gains: the experiment shows that BM25 is already a strong baseline in this domain (MRR = 0.835), and the incremental hybrid benefit is real but modest. The bootstrap confidence interval [0.003, 0.033] is narrow, suggesting the measurement is reliable rather than a large-sample artifact. This confirms the conventional NLP wisdom of "always benchmark BM25 first" and frames embedding-augmented retrieval as a consistent but marginal improvement over a strong lexical baseline.

**C2: Competitive Intent Classification Without Language Model Training.** The H2 result demonstrates that a frozen multilingual sentence encoder paired with gradient-boosted trees achieves Macro-F1 = 0.722 on an 8-class PT-BR intent classification task, training in approximately 6 seconds (LightGBM on CPU) with no encoder fine-tuning step. Embedding generation uses GPU-accelerated inference on Google Colab (Tesla T4), demonstrating that the entire pipeline is reproducible on freely available cloud infrastructure without dedicated ML hardware. This positions the "frozen encoder + LightGBM" architecture as a pragmatically accessible paradigm for organizations without the infrastructure or labeled data volume required for full fine-tuning. The result is particularly relevant for the Brazilian customer service context, where dedicated GPU deployment remains costly and annotation budgets are limited. The finding complements and extends recent work demonstrating that strong pretrained representations can transfer effectively to classification tasks with only linear or shallow nonlinear adaptation (Reimers and Gurevych, 2019; Tunstall et al., 2022).

**C3: Investigation of Deterministic Rules as Machine Learning Complement.** The H3 experiment provides the first empirical comparison, within a unified PT-BR conversational pipeline, of three rule integration strategies: rules-only, rules-as-features, and rules-as-override. The finding that rules-as-override degrades performance significantly (p = 8.2 × 10^-5, Macro-F1 = 0.648 versus 0.722 for ML-only) is a negative result of practical importance: it warns against the intuitive but incorrect design of using high-confidence rule activations to "veto" learned model predictions. The rules-as-features strategy, by contrast, shows no catastrophic degradation and a positive trend (+1.8 pp), warranting further investigation with a more comprehensive rule set and semantic predicate families. The current experiment was constrained to two lexical rules; the semantic predicate families (intent_score, embedding_similarity, contextual recurrence) implemented in the TalkEx rule engine were not exercised in H3, leaving the most promising integration pathway untested. This limitation is a planned extension.

**C4: TalkEx — A Modular, Auditable Conversation Intelligence Framework.** Beyond the empirical results, this thesis delivers TalkEx, an open-source Python framework implementing the full NLP pipeline: ingestion, turn segmentation, sliding context window construction, multi-level embedding generation, BM25 and ANN hybrid retrieval with score fusion, a supervised classification pipeline supporting logistic regression, MLP, and LightGBM, and a DSL-based semantic rule engine that compiles human-readable rules to typed ASTs with traceable per-prediction evidence. The framework supports four predicate families (lexical, semantic, structural, contextual), contamination-aware evaluation splits, and a unified experiment runner for reproducible hypothesis testing across five random seeds. The architectural separation of online inference from offline rule development and model training reflects design choices informed by production call-center deployment constraints. This architectural contribution is independent of the empirical results and remains valid regardless of the H3 or H4 verdicts.

**C5: Quantitative Evidence That Cascaded Inference Requires Genuine Stage Cost Differential.** The H4 refutation provides a concrete illustration of a design trap in pipeline optimization: when both stages of a cascade share pre-computed intermediate representations (embeddings in this case), the computational cost advantage of the "cheap" stage can vanish entirely. The measured result — Stage 1 cost (0.091 ms/window) exceeding Stage 2 cost (0.083 ms/window) — is a direct consequence of the embedding pre-computation being loaded into both pipelines' feature extraction steps. This finding generalizes to any cascaded system where stages share upstream representations and can be useful to practitioners designing latency-sensitive multi-stage classifiers. The path to realizing cascade benefit, as identified in Section 7.4, requires genuine architectural separation with a lexical-only fast path that never loads embedding representations.

## 7.3 Limitations

A rigorous account of this thesis must enumerate the conditions under which its findings may not generalize, and the methodological choices that constrain interpretation.

**1. Synthetic Data Dominance.** Approximately 60% of the 2,122 post-audit records were generated by a large language model and then human-reviewed to confirm intent label correctness (audit confirmation rate >= 96.7%). While the review process mitigated mislabeling, LLM-generated conversations exhibit stylistic uniformity, vocabulary overlap with common training corpora, and reduced spontaneous disfluency compared to real customer service transcripts. All performance figures reported in this thesis should be interpreted as upper bounds that may not transfer directly to real call-center data. No external real-world validation set was available at the time of writing.

**2. Single Domain and Single Language.** All experiments were conducted on PT-BR customer service conversations from a single synthetic dataset (RichardSakaguchiMS/brazilian-customer-service-conversations). No evaluation was performed across domains (e.g., banking vs. telecommunications vs. retail), languages, or channel types (voice with ASR noise, email, chat). Generalization claims beyond this single condition are not supported by the present evidence.

**3. No Cross-Domain Evaluation Completed.** The leave-one-domain-out (LODO) and stratified k-fold experiments designed to produce genuine confidence intervals and cross-domain generalization estimates were implemented (scripts: run_kfold_experiment.py, run_lodo_experiment.py) but not executed prior to the submission deadline. The hypothesis verdicts therefore rely on a single 80/20 stratified train-test split rather than on aggregated cross-validation estimates, and confidence intervals over macro-F1 are not available from the classification experiments.

**4. Dataset Scale Below Original Target.** The post-audit dataset contains 2,122 records against a planning target of approximately 3,500. The reduced scale limits statistical power in the smaller intent classes (saudacao, compra) and increases variance in per-class F1 estimates. The class with the lowest F1 (compra, F1 = 0.488) is also among the smallest, suggesting label noise or feature insufficiency that additional data could disambiguate.

**5. Researcher-Defined Rule Set.** The two rules deployed in H3 (rule_cancel and rule_complaint) were authored by the thesis researcher, not by domain experts (e.g., call-center supervisors, compliance analysts). This introduces potential rule design bias: the researcher may unconsciously write rules that overlap with the classifier's learned features rather than complementing its blind spots. Domain expert co-authorship is essential for production rule deployment and for a fair evaluation of the rules-as-features paradigm.

**6. Weak Supervision via Label Propagation.** Context windows inherit the intent label of the parent conversation rather than receiving window-level annotation. A 5-turn window may span the transition between two intents or include turns where the customer's stated intent has already been resolved. This form of weak supervision introduces label noise at the window level that is not modeled in the evaluation.

**7. Clean Text Only — No ASR Noise.** The pipeline was evaluated exclusively on clean transcribed text. Real voice channel input arrives through automatic speech recognition, introducing substitution errors, deletion errors, and disfluency artifacts that degrade lexical matching (BM25 and TF-IDF features) more severely than embedding features. The relative contributions of lexical versus embedding features, as measured in the ablation study, are likely to shift in the presence of ASR noise, potentially amplifying the case for semantic representations.

**8. Rule Coverage Insufficient for H3 Evaluation.** The TalkEx rule engine implements 11 modules with four predicate families, including semantic predicates (intent_score, embedding_similarity) and contextual predicates (repeated_in_window, occurs_after). Only 2 lexical rules were evaluated in H3. The semantic and contextual predicate families, which are architecturally the most novel, were untested. The H3 inconclusive verdict must therefore be understood as a verdict on lexical-only rule integration, not on the full semantic rule integration paradigm.

**9. Frozen Encoder Not Compared to Fine-Tuned Baseline.** C2 reports competitive performance from the frozen paraphrase-multilingual-MiniLM-L12-v2 encoder. No comparison was made against a fine-tuned version of the same or larger encoder (e.g., via supervised contrastive learning or few-shot prompting). Whether fine-tuning would close the remaining accuracy gap on challenging classes (compra, saudacao) is unknown. The frozen-first architecture is a principled cost-reduction choice, not a claim that fine-tuning is unnecessary.

**10. High Per-Class F1 Variance.** The full pipeline's per-class F1 ranges from 0.488 (compra) to 0.946 (cancelamento). This range of 0.46 F1 points indicates that macro-F1 averaging masks substantially unequal classification quality. Practical deployment of the system in a setting where all 8 intent classes have equivalent business value would require class-specific threshold calibration and potentially targeted data collection for the weakest classes. Brier score and ECE (expected calibration error) metrics are implemented in the pipeline but were not analyzed at the time of writing.

## 7.4 Future Work

The experimental findings and the architecture's current gaps point to several well-defined research and engineering directions.

**Fine-Tuning and Contrastive Learning.** The most direct path to closing the accuracy gap on semantically similar intents (duvida_produto vs. duvida_servico; compra vs. cancelamento) is supervised fine-tuning of the sentence encoder on conversation-level contrastive pairs. A frozen-to-fine-tuned ablation using the same LightGBM downstream classifier would isolate the representation quality effect and determine whether the "frozen encoder + LightGBM" paradigm is a near-optimal configuration or a practical approximation of a more capable fine-tuned system. The infrastructure for this experiment (finetune_baseline.py) has been implemented and is pending execution.

**External Validation with Real Production Data.** The most important open question is whether the performance figures generalize beyond LLM-generated conversations. A partnership with a Brazilian customer service operation to obtain a sample of real, anonymized transcripts — even a few hundred conversations — would provide the first genuine out-of-distribution validation of the pipeline. The LODO protocol is already implemented and parameterized to accept any domain split, making this a low-engineering-effort experiment once data access is available.

**Semantic and Contextual Rule Evaluation.** H3 evaluated only two lexical predicates. The semantic predicate families — particularly intent_score (which fires the rule engine when the classifier confidence for a target class exceeds a threshold) and embedding_similarity (which tests cosine proximity to a reference embedding) — are architecturally distinct from lexical matching and more likely to complement the LightGBM model's existing feature space. A dedicated experiment varying rule predicate type (lexical vs. semantic vs. contextual) while holding rule count constant would decompose the H3 result and potentially confirm the original hypothesis with a richer rule set.

**Cost-Differentiated Cascade Architecture.** H4's refutation identifies the remedy: the fast Stage 1 path must not load or invoke embedding computation. A genuine two-stage design would use a lexical-only logistic regression classifier with no embedding dependency as Stage 1 (estimated latency: 1-5 ms per window on CPU), escalating to the full LightGBM + embedding pipeline as Stage 2 (estimated latency: 40-80 ms per window, dominated by encoder inference). This design preserves the cost differential that H4 required. A re-implementation of H4 with this clean architectural separation is the most direct path to testing the cascade hypothesis under valid conditions.

**Cross-Domain Evaluation via LODO Protocol.** The leave-one-domain-out experiments (run_lodo_experiment.py) are implemented and ready to execute. They would estimate the domain generalization gap — the difference between within-domain and cross-domain classification performance — and test whether the hybrid pipeline generalizes better than lexical-only models across product domain shifts (e.g., a model trained on service and billing conversations evaluated on retail and technical support conversations). This is the highest-priority pending experiment for the dissertation's generalization claim.

**Calibration Analysis.** Brier score and ECE computation are implemented in the pipeline. Analyzing classifier calibration is prerequisite for threshold-based cascade design (H4), confidence-gated rule invocation (H3 semantic predicates), and abstention mechanisms for low-confidence predictions. The empirical cluster analysis of embedding cosine similarity during data audit identified a natural boundary at approximately 0.74-0.75 for within-class cohesion; this suggests an empirically motivated abstention threshold that calibration analysis could formalize.

**Intent Discovery via Offline LLM Clustering.** A complete conversation intelligence system requires not only classification of known intents but also discovery of emerging contact reasons. The offline intent discovery pipeline — clustering over context window embeddings, LLM-assisted label generation, taxonomy promotion with human review — is architecturally designed and documented in TalkEx but was not experimentally evaluated in this thesis. This represents the natural extension toward a self-updating system.

## 7.5 Closing Remarks

This thesis set out to determine whether a hybrid cascaded architecture combining lexical retrieval, frozen dense embeddings, and deterministic semantic rules outperforms isolated paradigms for intent classification in PT-BR customer service conversations. The answer is nuanced, as responsible experimental science demands.

Two findings are robust and practically actionable. First, hybrid retrieval provides consistent, statistically significant gains over BM25 alone, confirming that dense embeddings capture complementary signal in the conversational domain. The gains are real but modest; BM25 is a strong baseline that should always be measured before investing in semantic retrieval infrastructure. Second, and more substantially, frozen multilingual sentence embeddings transform classification performance in a way that no lexical feature engineering achieves: the +33 percentage point macro-F1 contribution from embeddings in the ablation study makes clear that dense representations are not a luxury but a necessity for this task. Achieving Macro-F1 = 0.722 with no training of any neural component and a 6-second LightGBM fit — using only Google Colab's free-tier GPU for embedding inference — establishes a practically accessible ceiling for the "frozen encoder + gradient boosting" paradigm.

Two findings are negative or inconclusive — and they are scientifically valuable precisely because they are honest. The cascade design as implemented does not deliver cost reduction, because the structural prerequisite (genuine stage cost differential) was not satisfied. Future cascade designs must enforce architectural separation from the first line of code. The rule engine's marginal contribution remains unmeasured for the predicate families most likely to be useful; the current H3 experiment evaluated a configuration too sparse to discriminate signal from noise.

The broader message of this work is one of pragmatic accessibility. Competitive conversation analysis — retrieval, multi-class classification, auditable decision rules — is achievable without dedicated GPU infrastructure (Google Colab's free-tier suffices), without large language model inference in the critical path, and without the labeled data volumes typically associated with production NLP systems. For organizations deploying customer service analytics in resource-constrained environments, particularly in Portuguese and other languages underrepresented in large model pretraining, the frozen-encoder paradigm offers an accessible and interpretable entry point. The TalkEx framework, with its typed data models, versioned predictions, and DSL-based rule engine, provides the architectural foundation to grow from this starting point toward a production-grade system as data and infrastructure scale.

This thesis is an exploratory investigation with preliminary empirical validation. It establishes baselines, confirms two architectural choices, identifies one design failure, and leaves the most architecturally interesting question — whether semantic rules genuinely complement statistical learning in conversational intent classification — as the open and compelling problem for future work.

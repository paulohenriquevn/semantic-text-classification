# Changelog

All notable changes to the Semantic Conversation Intelligence Engine will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/),
and this project adheres to [Semantic Versioning](https://semver.org/).

## [Unreleased]

### Added
- Project scaffolding with `src/` layout, `pyproject.toml`, and quality gate tooling (no-ticket)
- Domain exception hierarchy: `EngineError`, `EngineValidationError`, `PipelineError`, `ModelError`, `RuleError`, `ConfigurationError` (no-ticket)
- Domain enums: `SpeakerRole`, `Channel`, `ObjectType`, `PoolingStrategy` using Python 3.11 `StrEnum` (no-ticket)
- Domain type primitives: `ConversationId`, `TurnId`, `WindowId`, `EmbeddingId`, `PredictionId`, `RuleId`, `Score`, `Vector` (no-ticket)
- Core data model `Conversation` — complete interaction with channel, temporal, and operational metadata (no-ticket)
- Core data model `Turn` — individual utterance with speaker attribution and offset validation (no-ticket)
- Core data model `ContextWindow` — sliding window of adjacent turns with structural audit fields (no-ticket)
- Core data model `EmbeddingRecord` — versioned dense vector with `dimensions == len(vector)` invariant (no-ticket)
- Core data model `Prediction` — classification result with score/confidence/threshold ranges and `is_above_threshold` property (no-ticket)
- Core data model `RuleExecution` — auditable rule evaluation with typed `EvidenceItem` and `matched=True → evidence required` invariant (no-ticket)
- ADR-001: Package layout and public API (`src/` layout, `semantic_conversation_engine` package) (no-ticket)
- ADR-002: Frozen and strict pydantic models (`ConfigDict(frozen=True, strict=True)`) (no-ticket)
- ADR-003: Embedding vectors as `list[float]` in data models, `ndarray` at computation boundaries (no-ticket)
- ADR-004: ContextWindow structural fields for auditability and reproducibility (no-ticket)
- Input contracts: `TranscriptInput`, `SpeakerHint`, `QualitySignals` boundary objects and `SourceFormat` enum for pipeline ingestion (no-ticket)
- Pipeline stage protocols: `Segmenter` and `ContextBuilder` as `typing.Protocol` interfaces (no-ticket)
- Configuration objects: `SegmentationConfig` and `ContextWindowConfig` with frozen/strict validation (no-ticket)
- Text normalization: Unicode NFKC, whitespace collapsing, and line stripping with per-flag control (no-ticket)
- Turn parser: labeled (SPEAKER: text), multiline (line-per-turn), and plain (single block) format support (no-ticket)
- Merge/filter heuristics: consecutive same-speaker merging and min_turn_chars filtering (no-ticket)
- Lexical feature extraction: char_count, word_count, has_question, line_count, avg_word_length (no-ticket)
- `TurnSegmenter` orchestrator: end-to-end segmentation pipeline implementing the `Segmenter` protocol (no-ticket)
- Sliding window generation: configurable window_size, stride, partial tail handling, and min_window_size constraint (no-ticket)
- Window text rendering: speaker labels, custom delimiters, normalized_text preference with raw_text fallback (no-ticket)
- Role-aware view extraction: customer_text and agent_text in window metadata without redundant speaker labels (no-ticket)
- Window operational metrics: speaker_distribution, turn counts, word/char counts in window metadata (no-ticket)
- `SlidingWindowBuilder` orchestrator: end-to-end context window construction implementing the `ContextBuilder` protocol (no-ticket)
- Namespaced metadata convention: `role_views` and `speakers` namespaces in window metadata for collision-free pipeline governance (no-ticket)
- Build-level coverage metrics: `compute_build_coverage()` with orphan turn detection, coverage ratio, and multi-window overlap analysis (no-ticket)
- `TextProcessingPipeline` orchestrator: end-to-end pipeline composing `Segmenter` + `ContextBuilder` with structured warnings and operational stats (no-ticket)
- `PipelineResult` execution envelope: frozen result carrying conversation, turns, windows, coverage, warnings, and stats (no-ticket)
- `PipelineWarning` structured warnings: `NO_TURNS_PRODUCED`, `NO_WINDOWS_PRODUCED`, `ORPHAN_TURNS_DETECTED` with machine-readable codes and diagnostic context (no-ticket)
- Integration tests for full text processing pipeline: end-to-end flow, determinism, coverage, warnings, and format support (no-ticket)
- Embedding boundary objects: `EmbeddingInput` and `EmbeddingBatch` for embedding generation requests with source identity and metadata (no-ticket)
- Embedding configs: `EmbeddingModelConfig` (model identity, pooling, normalization) and `EmbeddingRuntimeConfig` (cache, retries, timeout) (no-ticket)
- Retrieval configs: `LexicalIndexConfig` (BM25 k1/b), `VectorIndexConfig` (metric, index type, dimensions), `HybridRetrievalConfig` (fusion strategy, reranker, filters) (no-ticket)
- Retrieval enums: `DistanceMetric`, `IndexType`, `FusionStrategy`, `QueryType`, `RetrievalMode` (no-ticket)
- Retrieval data types: `RetrievalFilter`, `RetrievalQuery`, `RetrievalHit`, `RetrievalResult` with provenance scores and graceful degradation mode (no-ticket)
- Pipeline protocols: `EmbeddingGenerator`, `VectorIndex`, `LexicalIndex`, `HybridRetriever`, `Reranker` as `typing.Protocol` interfaces (no-ticket)
- Embedding preprocessing: task-specific prefixing (E5 `passage:`/`query:`), object type tags (`[TURN]`/`[WINDOW]`/`[CONVERSATION]`), and token count estimation (no-ticket)
- Pooling strategies: `mean_pool`, `max_pool`, `l2_normalize`, and `apply_pooling` dispatcher with strategy validation (no-ticket)
- `NullEmbeddingGenerator`: deterministic hash-based fake embeddings for testing full pipeline without ML dependencies (no-ticket)
- `SentenceTransformerGenerator`: real embedding generation with conditional import, batch encoding, and pooling integration (no-ticket)
- `GenerationStats` observability: batch size, wall-clock time, and model name tracked per generate() call (no-ticket)
- `EmbeddingCache`: LRU-bounded in-memory cache keyed by (object_id, model_name, model_version, pooling_strategy) with hit rate tracking (no-ticket)
- `InMemoryBM25Index`: pure-Python BM25 Okapi lexical index with configurable k1/b, case-insensitive tokenization, incremental indexing, and operational stats (no-ticket)
- `InMemoryVectorIndex`: numpy-based flat exact-search vector index supporting cosine, L2, and dot-product metrics with save/load persistence (no-ticket)
- Index document builders: `context_window_to_lexical_doc`, `context_windows_to_lexical_docs`, `embedding_record_to_hit_metadata` for domain-to-index adaptation (no-ticket)
- Integration tests for full index pipeline: ContextWindow → EmbeddingGenerator → VectorIndex → search, and ContextWindow → BM25 → search (no-ticket)
- Score fusion strategies: `reciprocal_rank_fusion` (RRF) with configurable k parameter and `linear_fusion` with min-max normalization and semantic weight control (no-ticket)
- `SimpleHybridRetriever`: orchestrator combining lexical + semantic search with automatic mode routing (HYBRID/LEXICAL_ONLY/SEMANTIC_ONLY), graceful degradation, and configurable fusion (no-ticket)
- Hybrid retrieval observability: stats include `lexical_candidates`, `semantic_candidates`, `union_candidates`, `fusion_strategy`, `retrieval_mode`, `hybrid_latency_ms` (no-ticket)
- Integration tests for full hybrid retrieval pipeline: raw text → pipeline → dual index → hybrid search with RRF/LINEAR fusion strategies, mode routing, and top-k enforcement (no-ticket)
- Evaluation dataset schema: `EvaluationDataset`, `EvaluationExample`, `RelevanceJudgment` with JSON save/load round-trip for reproducible benchmarks (no-ticket)
- IR evaluation metrics: `recall_at_k`, `precision_at_k`, `reciprocal_rank` (MRR component), `ndcg` with graded relevance support (no-ticket)
- `BenchmarkRunner`: retriever-agnostic evaluation runner with multi-K metrics, per-query detail, and aggregated results across multiple retrieval strategies (no-ticket)
- `ExperimentReport` with JSON and CSV export: comparison reports for BM25 vs VECTOR vs HYBRID retrieval methods (no-ticket)
- Integration tests for evaluation benchmark: end-to-end pipeline → evaluation dataset → multi-retriever comparison with JSON/CSV serialization (no-ticket)
